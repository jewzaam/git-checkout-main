# GCM - Git Checkout Master

Git Checkout Master (GCM) is a portable Python git workflow automation tool that manages the "checkout master" workflow with proper fork management and branch cleanup.

## Key Files
- **Main executable**: `gcm.py` (325 lines) - Single-file design for portability
- **Configuration template**: `.gcm.yaml.example` - Copy to `.gcm.yaml` for local config
- **Build system**: `Makefile` + `make/*.mk` - Modular makefiles for development workflow

## Development Workflow

### Setup Environment
```bash
make venv && make uv && make requirements-dev
```

### Testing (Fast - Never Cancel)
- `make test-unit` - 43 unit tests (1 second)
- `make test-integration` - 12 integration tests (2 seconds) 
- `make test-all` - All 55 tests (2 seconds)

### Code Quality
- `make lint` - Format and lint with ruff (0.3 seconds)

### Application Usage
- `./gcm.py --help` - Show usage
- `./gcm.py --dry-run` - Preview actions without changes

## Project Structure
- `/tests/unit/` - Unit tests with mocking
- `/tests/integration/` - Integration tests with real git repos
- `/tests/helpers/` - Test utilities and factories
- `/make/` - Modular makefile components
- `/docs/` - Documentation
- `/reference/` - Original implementation reference

## Key Dependencies
- **Python 3.8+** with PyYAML for configuration
- **pytest** for testing with 64% coverage
- **ruff** for linting/formatting

## Configuration Search Order
1. `.gcm.yaml` (local repository)
2. `~/.gcm.yaml` (user home)
3. `~/.config/gcm.yaml` (XDG config)

## Error Handling
- May fail with "Could not determine trunk branch" outside git repos (expected)
- All git operations have comprehensive error handling
- Configuration errors show helpful messages
